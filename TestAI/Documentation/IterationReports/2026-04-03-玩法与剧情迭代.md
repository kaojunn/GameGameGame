# 迭代报告：2026-04-03 — 玩法与剧情（自我迭代）

## 1. 迭代前玩法（摘要）

- **核心**：节点图 + 选项推进；四维、等级、经验；装备三槽 + 背包拖放；消耗品使用区；检定为属性 + 1d10。
- **反馈**：叙事日志、右侧状态文本；**缺少**「本局读了多少节点、点了几次选项」的显式进度感（不利于多结局对比与复盘）。

## 2. 外部趋势要点（简要）

| 来源语境 | 要点 |
|----------|------|
| 分支叙事 / 互动小说讨论 | 玩家更在意**选择是否像有重量**（后果可感知、可复盘），而不只是单次阅读体验。 |
| 独立叙事作品实践 | **多路线、隐藏条件**（物品/属性门槛）增强重玩动机；短支线可低成本扩宽世界感。 |
| 工具与制作流 | 可视化节点、分支维护成本下降；小步增加**可验证支线**比一次大改剧情更安全。 |

（非学术论文；用于选题语境，具体实现以本仓库代码为准。）

| 方向 | 与当前差距 | 本次是否采纳 |

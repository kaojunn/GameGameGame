# 迭代报告：2026-04-02 — NorthernTown2026 结局图鉴（跨周目）

## 1. 迭代前玩法（当时是什么样）

- **玩家操作**：点击选项推进剧情；在右侧 UI 中拖拽物品卡片到装备槽/使用区/背包。
- **目标与推进**：通过分支选项与属性检定（属性 + 1d10）走向不同剧情节点，最终到达若干结局。
- **失败/胜利形态**：无传统“战斗失败画面”；以到达不同结局节点作为单局收束。
- **进度/Meta**：
  - 有单局进度：已读节点数、已做选择数（周目重置清零）。
  - 有成长：等级、经验、属性、装备加成。
  - **缺口**：多结局重玩时没有“跨周目结局收集反馈”，玩家难以判断自己还差哪些路线。
- **反馈**：叙事日志 + 右侧状态文本；无独立音效系统、无结局收集统计展示。

## 2. 趋势要点（外部检索浓缩）

1. **互动小说社区常见做法：记录已达成结局，鼓励重玩复盘**  
   语境：Twine/IF 社区讨论“跨周目追踪已达成结局”；Inform 生态里有 Recorded Endings 扩展用于记录并查看结局。  
   参考：  
   - https://intfiction.org/t/tracking-which-endings-have-been-achieved-across-playthroughs/46673  
   - https://emshort.blog/2009/05/08/inform-7-extension-recorded-endings/

2. **“Choices Matter / Multiple Endings”是平台可见标签，结局差异是叙事卖点**  
   语境：Steam 里存在“Choices Matter”“Multiple Endings”标签页和专题策展。  
   参考：  
   - https://store.steampowered.com/tags/en/Choices+matter  
   - https://store.steampowered.com/tags/en/Multiple+Endings  
   - https://store.steampowered.com/curator/45038463-Multiple-Endings-Choices/

3. **选择体验设计强调“后续提醒/可见后果”**  
   语境：选择设计实践常强调让玩家能看到自己决策累计结果，否则“选择感”会变弱。  
   参考：  
   - https://www.gamedeveloper.com/design/meaningful-choice-in-games-practical-guide-case-studies  
   - https://www.choiceofgames.com/2016/12/how-to-write-intentional-choices/

## 3. 对比结论（为什么只选这一项）

| 方向 | 与当前项目差距 | 本次小迭代适配性 |
|---|---|---|
| A. 结局图鉴（跨周目持久化） | 当前无结局收集 Meta | **采纳**：改动集中在状态与引擎，验收标准清晰 |
| B. 选项后果预告（按钮上显示风险/奖励） | 需改选项模型与按钮布局逻辑 | 暂不做：UI与叙事字段联动面较大 |
| C. 分支可视化地图（路线图） | 需新增完整 UI 模块与节点映射 | 暂不做：依赖多、非“小改动” |

**最终只选 A：结局图鉴**。  
理由：不改变现有剧情结构，不影响拖拽装备系统，且可明确验收“首次达成新结局时是否入档、状态栏是否累计显示”。

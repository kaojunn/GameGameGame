# NorthernTown2026 迭代报告（2026-04-03）：检定提示可视化

## 迭代前玩法（当时是什么样）

- 项目是运行时构建 UGUI 的文字冒险流程：节点文本 + 选项按钮推进。
- 检定规则已存在：`总值 = 对应属性 + 1d10`，与阈值比较决定成功/失败分支。
- 但在点击前，玩家只能看到选项文案，看不到当前属性下的大致通过率；检定结果要点下去后才在日志里揭示。
- 装备与消耗品会影响属性，所以玩家会频繁做“当前要不要尝试检定”的决策，但缺少即时反馈。

## 趋势要点（外部检索的浓缩）

1. **对话检定信息前置（BG3 语境）**  
   BG3 的骰子相关资料强调：在需要检定时，玩家会看到与检定相关的信息（难度、加值、成功概率/结果预期），帮助玩家在提交选择前评估风险。  
   参考语境：`bg3.wiki Dice rolls` 页面关于检定、DC、概率显示的说明。

2. **“把概率做成对话选项的一部分”（Disco Elysium 语境）**  
   Disco Elysium 开发日志提到：主动检定在对话中以“特殊选项”出现，并通过 tooltip 告知成功机会，让玩家把对话当成一次策略决策。  
   参考语境：官方 devblog《ON ACTIVE SKILL CHECKS》关于“special dialogue option + chances of succeeding”。

3. **可感知后果提升选择意义（叙事设计语境）**  
   意义选择框架里，“Awareness” 是关键组成：玩家要明确知道自己在做什么决策、可能付出什么代价。  
   参考语境：GameDeveloper 文章《Meaningful Choice in Games》。

## 对比结论（为什么选这一项）

| 范式/趋势 | 当前项目差距 | 是否适合这次小迭代 |
|---|---|---|
| 对话检定前置信息（显示概率/门槛） | 现有检定只在结算后可见，点击前不可评估风险 | **是**：仅改动按钮文案生成，依赖少，验收直接 |
| 检定类型分层（可重试/不可重试） | 当前无白/红检定语义与状态持久化 | 否：需要新增字段与内容层规则，改动面较大 |
| 选择后长期提醒（关系/世界反馈） | 当前有日志但缺少跨节点持续提醒系统 | 否：需要剧情数据与状态系统联动，不适合小步 |

本次只选 **“检定提示可视化”**：  
在不改剧情图结构、不增新资源的前提下，给检定选项追加“当前成功率 xx%”提示，提升决策可读性。

## 本次改动列表（改了哪些文件/行为）

1. `Assets/NorthernTown2026/Scripts/TextAdventureEngine.cs`
   - 新增 `FormatChoiceLabel(ChoiceOption choice)`：
     - 普通选项：返回原文案。
     - 检定选项：拼接 `【当前成功率 xx%】`。
   - 新增 `ComputeCheckSuccessRatePercent(int statValue, int threshold)`：
     - 按当前实现的 1d10 等概率规则计算成功面数并转为百分比（10% 精度）。
     - 使用 `Mathf.Clamp` 保证范围 0%~100%。

2. `Assets/NorthernTown2026/Scripts/TextAdventureBootstrap.cs`
   - 在 `RebuildChoiceButtons()` 中，按钮文本由 `opt.Text` 改为 `_engine.FormatChoiceLabel(opt)`。
   - 结果：有检定的选项会实时显示成功率，无检定选项不变。

3. `Assets/NorthernTown2026/GAMEPLAY_FRAMEWORK.md`
   - 文档同步：补充“检定选项显示即时成功率提示”的规则说明，避免实现与文档脱节。

## 验收自检（逻辑推演）

- 进入含检定的节点（如 `station_enter`、`old_man_intro`、`courier_meet`）：
  - 对应选项应显示 `【当前成功率 xx%】`。
- 装备/卸下会改变属性：
  - 触发 `OnStateChanged` 后重建选项，成功率应随属性变化即时更新。
- 非检定选项：
  - 文案保持原样，不出现多余提示。
- 检定结算：
  - 仍沿用原 `属性 + 1d10 >= 阈值` 逻辑，分支结果无行为回归。

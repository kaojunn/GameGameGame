# 迭代报告：2026-04-03 — NorthernTown2026 结局图鉴小迭代

## 1) 迭代前玩法（当时是什么样）

- **玩家操作**：在叙事区阅读节点文本，通过底部按钮做选择；右侧可拖拽物品卡到装备槽 / 使用区 / 背包区。
- **目标与流程**：通过分支与检定推进到不同 `ending_*` 节点；可点击「再玩一次」回到 `start`。
- **失败/胜利判定**：无传统 HP 死亡；以到达不同结局作为单局终点。
- **进度/Meta**：
  - 本局内已有「已读节点 / 已做选择」计数；
  - 但**跨周目**仅能靠玩家记忆判断是否看过某结局，缺少图鉴式反馈。
- **反馈**：叙事日志会显示检定、经验、装备变更；状态面板显示等级、属性、装备、背包。

## 2) 趋势要点（外部检索浓缩）

1. **多结局标签与重玩驱动**  
   Steam 生态中「Multiple Endings」被广泛用于强调重玩价值，玩家通常会主动追求不同走向与收集式体验。  
   语境来源：Steam250 的 Multiple Endings 标签聚合页（https://steam250.com/tag/multiple_endings）。

2. **叙事引擎中的“已见/关键选择”可视化**  
   Narrat 3.3.8 在更新中加入 choice tracking（如已看过选项样式），核心思路是把“探索过什么”直接反馈给玩家。  
   语境来源：Narrat 官方更新日志（https://narrat.dev/news/narrat-3-3-8-release/）。

3. **短循环游戏强调 Action→Feedback→Motivation**  
   玩法反馈越及时，玩家越容易进入“再来一局”循环；叙事游戏同样可借此增强路线探索动力。  
   语境来源：关于 core loop/feedback loop 的公开设计文章讨论（如 Medium/UX 向综述，检索关键词 *Action Feedback Reward Motivation Repeat*）。

## 6) 验收自检（逻辑推演）

- 新会话启动后，图鉴总数应等于 `ending_` 前缀节点数；已解锁数来自 `PlayerPrefs`。
- 首次进入某个结局：
  - 日志出现「结局图鉴已解锁」提示；
  - 状态栏 `x/y` 增加 1；
  - 再次进入同一结局不重复增加。
- 选择「再玩一次」触发周目重置时：
  - 本局计数（节点/选择）归零；
  - 图鉴解锁进度保持（跨周目）。
- 键名冲突检查：统一使用前缀 `NorthernTown2026.EndingUnlocked.`，与现有玩法字段无重名。

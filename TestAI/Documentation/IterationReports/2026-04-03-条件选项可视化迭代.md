# 迭代报告：2026-04-03 — 条件选项可视化（禁用态 + 门槛提示）

## 1. 迭代前玩法（当时是什么样）

- **输入与操作**：`MainTextAdventure.unity` 场景仅挂 `TextAdventureBootstrap`，运行时生成 UGUI。玩家通过点击底部选项推进节点；右侧可拖拽装备卡到槽位、拖消耗品到使用区。
- **目标与胜负**：目标是通过分支选择到达不同结局节点（如 `ending_public / ending_leave / ending_hidden` 等）；到达结局并出现「再玩一次」视作一轮完成。无传统 HP 失败态，失败更多体现在检定失败分支或较弱结局。
- **进度 / Meta**：已有 `RunNodesVisitedCount` 与 `RunChoicesCount`（本局节点/选择计数），并在状态栏展示。
- **反馈**：叙事日志输出节点文本与系统消息（检定、升级、装备/使用结果）。  
  但在本次迭代前，`RequiresItemId`/`RequiresInsightSum` 不满足时，选项会被直接过滤隐藏，玩家不知道“还有哪些路径”和“缺了什么”。

## 2. 趋势要点（外部检索浓缩）

1. **条件选项显隐是互动叙事引擎的常见表达手段**  
   Ren'Py 官方文档的 `menu` 条目支持 `if condition` 控制选项显示，且文档明确可配置显示为禁用态（而非完全消失）的交互思路。  
   语境来源：Ren'Py docs（In-Game Menus / conditional choices）。

2. **分支后果可感知，会提升“选择有重量”的体验**  
   Ink 社区与教程文档持续强调条件文本与变量追踪，让玩家能看到先前选择如何影响后续可选路径。  
   语境来源：inkle/ink 文档生态（WritingWithInk 相关资料）与 Ink 教程材料。

3. **轻量、可持续追踪的进度反馈在重玩结构中价值高**  
   Steamworks 对 stats/achievements 的说明强调“持续统计 + 关键节点存储”可用于玩家反馈与多次会话追踪。  
   语境来源：Steamworks「Stats and Achievements」文档。

## 3. 对比结论（为什么只选这一项）

| 方向 | 与当前项目差距 | 适不适合放进这一次小迭代 |
|------|----------------|--------------------------|
| A. 条件选项“可见但禁用”，并附门槛提示 | 当前是直接隐藏，信息不透明 | **采纳**：改动集中在引擎与按钮渲染，验收标准明确 |
| B. 新增长期持久化成就/统计（PlayerPrefs） | 需扩展数据生命周期与展示入口 | 暂不做：跨局设计口径尚未定 |
| C. 引入更复杂条件系统（多变量组合、一次性 flags） | 需改模型字段与剧情数据规模 | 暂不做：改动面过大 |

**最终只选 A**：最小改动即可提升“可理解性”，且不破坏现有剧情图和存档结构。

## 4. 本次改动列表（文件 / 行为）

1. `TestAI/Assets/NorthernTown2026/Scripts/StoryModels.cs`
   - 新增 `ChoiceView`（`Option`、`CanChoose`、`LockedHint`），作为 UI 层展示模型。

2. `TestAI/Assets/NorthernTown2026/Scripts/TextAdventureEngine.cs`
   - 新增 `GetChoiceViewsForCurrentNode()`：不再丢弃条件不足项，而是返回可选状态与锁定原因。
   - 新增 `EvaluateChoiceAvailability(...)`：
     - 物品不足时给出「需要物品：xxx」；
     - 属性和不足时给出「需要洞察+机巧 ≥ n（当前 m）」。
   - `Choose(...)` 增加二次校验：即使 UI 侧误触也不会执行结算，并输出「条件不足」日志。
   - 保留 `GetChoicesForCurrentNode()` 兼容接口（内部基于 `ChoiceView` 过滤可选项）。

3. `TestAI/Assets/NorthernTown2026/Scripts/TextAdventureBootstrap.cs`
   - `RebuildChoiceButtons()` 改为遍历 `GetChoiceViewsForCurrentNode()`。
   - 条件不足项改为：
     - 按钮禁用（`interactable = false`）；
     - 样式置灰；
     - 文案第二行显示门槛提示（富文本小字）。
   - 可选项保持原有点击推进行为。

4. `TestAI/Assets/NorthernTown2026/GAMEPLAY_FRAMEWORK.md`
   - 更新规则描述：从“条件不满足即隐藏”改为“全部显示、条件不足置灰并提示”，并记录引擎层二次校验。
